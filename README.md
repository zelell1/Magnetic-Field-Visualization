# Магнитное поле цилиндрического соленоида

Интерактивное демо вычисляет и визуализирует распределение магнитного поля `B(x, y)` в плоскости, содержащей ось цилиндрического соленоида. Параметры `D`, `L`, `N`, `I`, размер расчетной области и детализация задаются в интерфейсе.

## Запуск

```bash
npm test
npm run serve
```

После запуска сервера откройте `http://localhost:5173`.

Проект не использует внешние npm-зависимости: демо работает на Canvas, а тесты запускаются встроенным `node --test`.

## Деплой

В репозитории есть GitHub Actions workflow `.github/workflows/deploy.yml`. При push в ветку `main` он запускает тесты и публикует статическое демо на GitHub Pages.

После первого успешного workflow демо будет доступно по адресу:

## Физическая модель

Соленоид рассматривается как набор одинаковых круговых витков радиуса

```text
R = D / 2
```

с центрами `x_i` на оси соленоида. Поле находится по принципу суперпозиции:

```text
B(x, y) = sum_i B_loop(x - x_i, rho) * w_i,    rho = |y|
```

где `w_i` - вес расчетного витка. Если `N` мало, каждый виток считается отдельно. Если `N` велико, соседние витки группируются в расчетные элементы с суммарным весом, чтобы сохранить произведение `N I` и сделать интерактивную визуализацию быстрой.

Для одного кругового витка используются стандартные выражения через полные эллиптические интегралы `K(m)` и `E(m)`. Пусть `xi = x - x_i`, `rho = |y|`,

```text
alpha^2 = (R + rho)^2 + xi^2
beta^2  = (R - rho)^2 + xi^2
m       = 4 R rho / alpha^2
```

Тогда компоненты поля в осевой плоскости:

```text
Bx = mu0 I / (2 pi alpha) *
     [ K(m) + (R^2 - rho^2 - xi^2) / beta^2 * E(m) ]

Br = mu0 I xi / (2 pi rho alpha) *
     [ -K(m) + (R^2 + rho^2 + xi^2) / beta^2 * E(m) ]

By = sign(y) * Br
```

На оси `rho = 0` используется предельная формула без эллиптических интегралов:

```text
Bx = mu0 I R^2 / (2 (R^2 + xi^2)^(3/2)),    By = 0
```

Эллиптические интегралы вычисляются численно квадратурой Гаусса-Лежандра на отрезке `[0, pi/2]`:

```text
K(m) = integral_0^(pi/2) dtheta / sqrt(1 - m sin^2(theta))
E(m) = integral_0^(pi/2) sqrt(1 - m sin^2(theta)) dtheta
```

В коде используется 28-точечная квадратура Гаусса-Лежандра. Она выбрана вместо табличной аппроксимации Абрамовица-Стигана, потому что напрямую интегрирует определения `K(m)` и `E(m)`, не требует внешних зависимостей и стабильно работает на всем интервале `0 <= m < 1`. Точность контролируется тестами: при `m = 0` получается `pi/2`, а при `m = 0.5` и `m = 0.9` значения совпадают с эталонными с относительной ошибкой меньше `1e-12`.

## Проверочные формулы

Для непрерывного конечного соленоида с плотностью витков `n = N / L` поле на оси:

```text
Bx_axis(x) = mu0 n I / 2 *
             [ (x + L/2) / sqrt(R^2 + (x + L/2)^2)
              -(x - L/2) / sqrt(R^2 + (x - L/2)^2) ]
```

В центре длинного соленоида:

```text
Bx(0) ~= mu0 N I / L
```

В дальней зоне на оси поле стремится к дипольной асимптотике:

```text
Bx(x) ~= mu0 N I R^2 / (2 |x|^3)
```

Именно эти выражения используются в тестах и в графике профиля на оси.

Для тестового примера в интерфейсе используется именно эта формула конечного соленоида:

```text
D = 0.20 m, L = 0.80 m, N = 120, I = 2.0 A
Bx_superposition(0,0) = 365.74 microT
Bx_axis(0)            = 365.74 microT
```

Здесь `Bx_superposition` считается суммированием расчетных витков, а `Bx_axis` - по аналитической формуле конечного непрерывного соленоида.

## Тест-кейсы

Файл `tests/physics.test.js` проверяет:

- предел эллиптических интегралов при `m = 0`;
- значения эллиптических интегралов при `m = 0.5` и `m = 0.9`;
- совпадение поля одного витка на оси с аналитической формулой;
- симметрию поля: `Bx(x, y) = Bx(x, -y)`, `By(x, y) = -By(x, -y)`;
- переход к формуле длинного соленоида при `L >> D`;
- совпадение суперпозиции расчетных витков с формулой конечного непрерывного соленоида на оси;
- сходимость по числу расчетных витков `maxSamples`;
- дипольную асимптотику в дальней зоне;
- смену направления поля при смене знака тока.

## Численные замечания

Идеально тонкий провод имеет сингулярность прямо на линии витка. В интерфейсе для устойчивой картинки используется малое сглаживание около проводника; оно влияет только на несколько пикселей рядом с витками. В тестах формулы проверяются вдали от сингулярности.

Для больших `N` визуализация по умолчанию группирует витки. Это не меняет суммарные ампер-витки `N I`, а ошибка убывает при увеличении параметра "витки в расчете".

Для базового набора параметров `D = 0.20 m`, `L = 0.80 m`, `N = 120`, `I = 2.0 A` сравнение с аналитической формулой конечного соленоида дает:

```text
maxSamples   Bx(0,0), microT   ошибка относительно Bx_axis(0)
8            361.22            1.23%
12           365.56            0.048%
20           365.76            0.0067%
60           365.74            0.00077%
90           365.74            0.00034%
```

Поэтому для демонстрации достаточно `maxSamples >= 12`, чтобы ошибка в центре была меньше 1%. Значение по умолчанию `90` выбрано с большим запасом: оно сохраняет интерактивность и практически совпадает с осевой аналитической формулой.

## Выводы

Полученная карта показывает почти однородное поле внутри длинного соленоида и заметное рассеяние поля около торцов. При увеличении `L/D` центральное поле приближается к `mu0 N I / L`; при удалении от соленоида поле быстро убывает как `1 / |x|^3`, что соответствует магнитному диполю.
